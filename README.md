# GOST_SIGNATURE

Программная реализация схемы электронной цифровой подписи **ГОСТ Р 34.10-2012** и хеш-функции **ГОСТ Р 34.11-2012 («Стрибог»)** на языке Python.

Реализация не использует сторонние криптографические библиотеки. Арифметика эллиптических кривых, формирование и проверка подписи, а также хеш-функция «Стрибог» реализованы средствами стандартной библиотеки Python.

## Возможности

- Формирование и проверка ЭЦП по ГОСТ Р 34.10-2012 на эллиптических кривых
- Самостоятельная реализация хеш-функции ГОСТ Р 34.11-2012 (Стрибог) с длиной хеша **256** и **512** бит
- Поддержка 12 наборов параметров эллиптических кривых: тестовые из Приложения А стандарта, наборы CryptoPro из RFC 4357 и наборы TC26 из RFC 7836 / RFC 9215
- Генерация ключевой пары для любого набора параметров
- Подписание и проверка произвольных файлов через командную строку
- Сохранение и загрузка ключей в читаемом текстовом формате (JSON)
- Полная валидация публичного ключа при загрузке: проверка диапазона координат, принадлежности кривой и порядка точки (q·Q = O)
- Защитные проверки: контроль диапазона ключа подписи, идентичности кривой при сложении точек, границ компонент подписи
- 9 групп встроенных тестов, включая контрольные примеры M1/M2 из Приложения А ГОСТ Р 34.11-2012 и примеры А.1/А.2 формирования подписи из ГОСТ Р 34.10-2012

## Описание схемы ЭЦП

ГОСТ Р 34.10-2012 — схема электронной цифровой подписи на основе арифметики в группе точек эллиптической кривой над конечным простым полем. Стойкость основана на сложности задачи дискретного логарифмирования в группе точек эллиптической кривой.

Кривая задается уравнением `y² ≡ x³ + a·x + b (mod p)` над полем F_p и базовой точкой G простого порядка q.

| Вариант | Длина q     | Длина хеша | Длина подписи |
|---------|-------------|------------|---------------|
| 256-бит | 256 бит     | 256 бит    | 64 байта      |
| 512-бит | 512 бит     | 512 бит    | 128 байт      |

### Генерация ключевой пары

1. Выбрать кривую с параметрами `p, a, b` и базовой точкой `G` порядка `q`.
2. Сгенерировать случайное целое `d` из диапазона `(0, q)` — ключ подписи.
3. Вычислить `Q = d·G` — ключ проверки подписи (точка на кривой).

Открытый ключ: **(curve, Q)**.  Закрытый ключ: **(curve, d)**.

### Формирование подписи

1. Вычислить `h = H(M)` — хеш сообщения соответствующей длины.
2. Привести к числу: `α = int(h)`, `e = α mod q`, при `e = 0` положить `e := 1`.
3. Выбрать случайное `k ∈ (0, q)`.
4. Вычислить `C = k·G`, `r = C.x mod q`. Если `r = 0` — вернуться к п. 3.
5. Вычислить `s = (r·d + k·e) mod q`. Если `s = 0` — вернуться к п. 3.
6. Подпись: пара **ζ = (r, s)**.

### Проверка подписи

1. Проверить `0 < r < q` и `0 < s < q`. Если нет — подпись неверна.
2. Вычислить `e = int(H(M)) mod q`, при `e = 0` положить `e := 1`.
3. Вычислить `v = e⁻¹ mod q`.
4. Вычислить `z₁ = s·v mod q` и `z₂ = -r·v mod q`.
5. Вычислить `C = z₁·G + z₂·Q`.
6. Вычислить `R = C.x mod q`.
7. Подпись верна тогда и только тогда, когда **R = r**.

### Сериализация подписи

Подпись хранится как конкатенация `r || s`, где каждая компонента занимает `L = ⌈q.bit_length() / 8⌉` байт в формате big-endian:

- 64 байта для 256-битной кривой (32 + 32);
- 128 байт для 512-битной кривой (64 + 64).

## Описание хеш-функции Стрибог

ГОСТ Р 34.11-2012 — функция хеширования с длиной хеша 256 или 512 бит, построенная на основе конструкции Меркла-Дамгора с функцией сжатия `g_N`, использующей внутренний 12-раундовый блочный шифр.

### Базовые преобразования

- **S** — нелинейная побайтовая подстановка через таблицу π (256 байт)
- **P** — перестановка 64 байт состояния согласно таблице τ
- **L** — линейное преобразование над GF(2): умножение каждого из восьми 64-битных подвекторов на матрицу A
- **X[k]** — побитовое сложение состояния с ключом
- **LPS** = L ∘ P ∘ S — основная композиция, применяемая в раундах шифра

### Внутренний блочный шифр E(K, m)

12 раундов вида `X[K_i] ∘ LPS` плюс финальное `X[K₁₃]`. Раундовые ключи `K_i` развертываются из исходного `K` по схеме `K_{i+1} = LPS(K_i ⊕ C_i)` с двенадцатью 512-битными итерационными константами `C₁..C₁₂`.

### Функция сжатия

`g_N(h, m) = E(LPS(h ⊕ N), m) ⊕ h ⊕ m`

### Процедура хеширования

1. **Инициализация**: `h := IV`, `N := 0⁵¹²`, `Σ := 0⁵¹²` (для 256-битного варианта IV состоит из байтов 0x01, для 512-битного — из нулей).
2. **Полные блоки**: пока в сообщении остается ≥ 64 байт — обрабатывается очередной блок справа налево, обновляются `h`, счетчик длины `N` и контрольная сумма `Σ`.
3. **Неполный блок и финализация**: остаток дополняется по правилу `0…0 ‖ 1 ‖ M` до 512 бит, затем выполняются три финальных вызова функции сжатия — основной, для счетчика `N` и для суммы `Σ`.

Реализация учитывает **поправку 2018 года** к стандарту: на шаге 3.4 в `Σ` суммируется дополненный блок `m`, а не его длина `|m|`.
Без этой поправки контрольный пример M2 из Приложения А не дает ожидаемого значения.

## Использование

### Генерация ключевой пары

```bash
python gost_signature.py keygen --paramset test-256 --out generated_keys/mykey
```

Команда создает два файла: `generated_keys/mykey.priv` (приватный ключ) и `generated_keys/mykey.pub` (публичный ключ). Промежуточные директории создаются автоматически.

### Подписание файла

```bash
python gost_signature.py sign \
  --in document.txt \
  --key keys/mykey.priv \
  --out document.sig
```

### Проверка подписи

```bash
python gost_signature.py verify \
  --in document.txt \
  --key keys/mykey.pub \
  --sig document.sig
```

Команда возвращает код выхода `0`, если подпись верна, и `1`, если подпись недействительна или формат подписи некорректен.

### Полный цикл

```bash
python gost_signature.py keygen --paramset tc26-512-A --out keys/key512
python gost_signature.py sign --in document.txt --key keys/key512.priv --out document.sig
python gost_signature.py verify --in document.txt --key keys/key512.pub --sig document.sig
```

### Использование других наборов параметров

```bash
python gost_signature.py keygen --paramset cryptopro-A --out keys/cp
python gost_signature.py keygen --paramset tc26-256-B --out keys/tc26b
python gost_signature.py keygen --paramset tc26-512-C --out keys/tc26c
```

Полный список доступных наборов — см. раздел «Наборы параметров».

## Запуск тестов

```bash
python gost_signature.py test
```

Запускает 9 групп встроенных тестов:

1. `math_utils` — корректность `mod_pow` и теста Миллера — Рабина, включая отсечение чисел Кармайкла.
2. `streebog` — контрольные хеши M1 и M2 из Приложения А ГОСТ Р 34.11-2012 для 256- и 512-битного вариантов.
3. `curve_params` — проверка всех встроенных наборов параметров: простота `p` и `q`, диапазоны коэффициентов, ненулевой дискриминант, принадлежность `G` кривой, выполнение условия `q·G = O`.
4. `validate_public_key` — проверка корректного публичного ключа и негативных сценариев.
5. `signature_known_256` — контрольный пример А.1 из ГОСТ Р 34.10-2012.
6. `signature_known_512` — контрольный пример А.2 из ГОСТ Р 34.10-2012.
7. `signature_lengths` — проверка длины сериализованной подписи: 64 байта для 256-битных наборов и 128 байт для 512-битных.
8. `edge_cases` — границы `r`, `s`; неверная длина подписи; недопустимое значение `d`; пустое сообщение; сообщения длиной 63, 64 и 65 байт.
9. `end_to_end` — полный цикл `keygen → sign → verify` на всех встроенных наборах параметров с проверкой отрицательных сценариев.

## Наборы параметров

| Имя           | Источник                                                        | Размер   |
|---------------|-----------------------------------------------------------------|----------|
| `test-256`    | Приложение А.1 ГОСТ Р 34.10-2012                                | 256 бит  |
| `test-512`    | Приложение А.2 ГОСТ Р 34.10-2012                                | 512 бит  |
| `tc26-256-A`  | RFC 7836, Appendix A.2; Weierstrass-представление               | 256 бит  |
| `tc26-256-B`  | OID: RFC 9215; параметры: RFC 4357 / CryptoPro-A                | 256 бит  |
| `tc26-256-C`  | OID: RFC 9215; параметры: RFC 4357 / CryptoPro-B                | 256 бит  |
| `tc26-256-D`  | OID: RFC 9215; параметры: RFC 4357 / CryptoPro-C                | 256 бит  |
| `cryptopro-A` | RFC 4357; id-GostR3410-2001-CryptoPro-A-ParamSet                | 256 бит  |
| `cryptopro-B` | RFC 4357; id-GostR3410-2001-CryptoPro-B-ParamSet                | 256 бит  |
| `cryptopro-C` | RFC 4357; id-GostR3410-2001-CryptoPro-C-ParamSet                | 256 бит  |
| `tc26-512-A`  | RFC 7836, Appendix A.1; Weierstrass                             | 512 бит  |
| `tc26-512-B`  | RFC 7836, Appendix A.1; Weierstrass                             | 512 бит  |
| `tc26-512-C`  | RFC 7836, Appendix A.2; Weierstrass-представление               | 512 бит  |

**Примечание о тестовых параметрах.** Наборы `test-256` и `test-512` взяты из Приложения А ГОСТ Р 34.10-2012 и предназначены исключительно для проверки корректности реализации. Они не должны применяться для реальной защиты информации.

**Примечание о TC26-256-B/C/D.** Эти наборы математически совпадают с CryptoPro-A/B/C из RFC 4357, но имеют отдельные идентификаторы параметров, описанные в RFC 9215. В коде они представлены отдельными объектами `Curve`, чтобы при сохранении ключа в файле фиксировалось именно выбранное пользователем имя параметров.

**Примечание о tc26-256-A и tc26-512-C.** В RFC 7836 эти кривые определены в форме Twisted Edwards, но стандарт также предоставляет их эквивалентное Weierstrass-представление, которое и используется в реализации.

## Формат файлов

### Файл приватного ключа (`.priv`)

JSON в читаемом формате:

```json
{
  "version": 1,
  "paramset": "test-256",
  "d": "7a929ade789bb9be10ed359dd39a72c11b60961f49397eee1d19ce9891ec3b28"
}
```

### Файл публичного ключа (`.pub`)

```json
{
  "version": 1,
  "paramset": "test-256",
  "qx": "7f2b49e270db6d90d8595bec458b50c58585ba1d4e9b788f6689dbd8e56fd80b",
  "qy": "26f1b489d6701dd185c8413a977b3cbbaf64d1c593d26627dffb101a87ff77da"
}
```

### Файл подписи (`.sig`)

Бинарный формат: конкатенация `r || s`, где каждая компонента — целое число в формате big-endian. Длина:

- 64 байта для 256-битной кривой;
- 128 байт для 512-битной кривой.

## Демонстрационные ключи и подписи

В проект включены готовые ключевые пары и подписи, которые получены запуском самой программы и служат для иллюстрации результатов работы (раздел «Результаты» отчёта). Их можно перегенерировать в любой момент командами `keygen` и `sign`.

### Каталог `generated_keys/`

Для каждого из набора параметров здесь лежат две пары ключей — основная (`key*`) и вспомогательная (`other*`). Вторая пара используется для демонстрации отрицательного сценария: подпись, сделанная одним ключом, не проходит проверку на ключе проверки другого.

| Базовое имя        | Набор параметров | Назначение                              |
|--------------------|------------------|-----------------------------------------|
| `key256` / `other256`             | `test-256`      | 256-битный тестовый набор               |
| `key512` / `other512`             | `test-512`      | 512-битный тестовый набор               |
| `key_cpa` / `other_cpa`           | `cryptopro-A`   | набор CryptoPro-A                       |
| `key_tc26_256_b` / `other_tc26_256_b` | `tc26-256-B` | набор TC26-256-B                        |
| `key_tc26_512_c` / `other_tc26_512_c` | `tc26-512-C` | набор TC26-512-C                        |

Каждое имя — это пара файлов `*.priv` (ключ подписи `d`) и `*.pub` (ключ проверки `Q`). Значения `d` сгенерированы случайно при создании пары, поэтому в конкретных файлах они отличаются от приведённых выше примеров формата.

### Подписи файла `test.txt`

Файлы `test_*.sig` — это подписи демонстрационного сообщения `test.txt`, сформированные соответствующими ключами `key*.priv`:

| Файл подписи              | Ключ подписи               | Длина     |
|---------------------------|----------------------------|-----------|
| `test_256.sig`            | `key256.priv`              | 64 байта  |
| `test_512.sig`            | `key512.priv`              | 128 байт  |
| `test_cpa.sig`            | `key_cpa.priv`             | 64 байта  |
| `test_tc26_256_b.sig`     | `key_tc26_256_b.priv`      | 64 байта  |
| `test_tc26_512_c.sig`     | `key_tc26_512_c.priv`      | 128 байт  |

Проверить любую из них можно соответствующим публичным ключом, например:

```bash
python gost_signature.py verify --in test.txt --key generated_keys/key256.pub --sig test_256.sig
```

Файл `test_tampered.txt` содержит изменённую копию `test.txt`; проверка любой подписи против него завершается отказом (код выхода `1`), что демонстрирует контроль целостности.

## Ограничения учебной реализации

Реализация предназначена для демонстрации алгоритма и проверки на контрольных примерах. В частности:

- не реализована постоянная-времени защита от тайминговых атак на скалярное умножение точек;
- не выполнялась криптоаналитическая проверка генератора случайных чисел вне рамок стандартного `secrets`;
- реализация поддерживает только Weierstrass-представление кривых; кривые в форме Twisted Edwards используются через эквивалентные Weierstrass-параметры из RFC 7836.
